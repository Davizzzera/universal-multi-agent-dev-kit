# Planning Skills

> Procedures for planning tasks.

---

## Purpose
This category contains execution procedures for tasks in the planning domain.

## When Used
Loaded when a task specifically targets planning requirements.

## Typical Owner Agents
These skills are typically assigned to agents in the planning category, or orchestration agents.

## Skills

| Skill | Primary Purpose | Typical Owner Agents | Validation Level |
|-------|-----------------|----------------------|------------------|
| [task-breakdown](task-breakdown/SKILL.md) | Break a broad request into clear, actionable subtasks. | universal-orchestrator, task-router, technical-lead | manual-and-automated |
| [implementation-plan](implementation-plan/SKILL.md) | Create a safe, scoped and ordered implementation plan. | universal-orchestrator, technical-lead, software-architect | manual-and-automated |
| [acceptance-criteria-writing](acceptance-criteria-writing/SKILL.md) | Define measurable acceptance criteria for a task or feature. | requirements-engineer, product-manager, qa-tester | manual-and-automated |
| [definition-of-done](definition-of-done/SKILL.md) | Define what must be true before a task can be considered complete. | final-reviewer, qa-tester, product-manager | manual-and-automated |
| [rollback-plan](rollback-plan/SKILL.md) | Plan how to revert or safely recover from a change. | technical-lead, devops-specialist, database-specialist | manual-and-automated |
| [technical-decision-record](technical-decision-record/SKILL.md) | Document important technical decisions and tradeoffs. | software-architect, technical-writer, technical-lead | manual-and-automated |
